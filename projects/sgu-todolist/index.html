01 — Problem & Solution

Why this project exists

Traditional task management tools have real limitations that modern cloud architecture can solve.

⚠ The Problems

⚙

Manual deployment complexity

Deploying 6 microservices manually is error-prone and time-consuming, leading to inconsistent environments and deployment failures.

🔄

No real-time collaboration

Traditional task managers require page refresh to see changes. Team members work with stale data, causing conflicts and missed updates.

🧠

No intelligent task management

Users manually categorize and prioritize every single task. No automation to reduce cognitive overhead or suggest classifications.

📦

Monolithic limitations

A single codebase blocks the entire team. Can't scale services independently, and one bug can take down everything.

✅ The Solutions

🚀

CI/CD with GitHub Actions

Automated: push → Docker build → ECR → ECS Fargate. 50% less deploy time with zero manual intervention.

⚡

WebSocket real-time updates

STOMP + Redis Pub/Sub broadcasts task changes instantly to all connected clients. No more stale data.

🤖

AI-powered task labeling

Flask ML service with scikit-learn auto-classifies labels and suggests priority levels with confidence scoring.

🏗

Microservices architecture

6 independent services communicating via Kafka. Each developer works on their own service, scale what you need.

02 — Overview

Four permission levels, one platform

Role-based access control ensures the right people have the right permissions for effective project management.

Owner

  • Full control over project
  • Cannot be removed
  • Manage all members & settings

Admin

  • Manage members (except Owner)
  • CRUD tasks & sections
  • Assign tasks to members

Member

  • Create & manage own tasks
  • Comment with attachments
  • Personal inbox

Viewer

  • Read-only access
  • View tasks & sections
  • View comments

07 — Auth & Security

JWT + OAuth2 + 2FA

Multi-layered security with dual-token architecture, social login, and two-factor authentication.

Token Architecture

  • Access Token (15 min) — carries userId, email, role
  • Refresh Token (7 days) — stored in httpOnly Cookie
  • HMAC-SHA256 signing with Base64 secret
  • Redis blacklist for secure logout

Security Features

  • OAuth2 Google login with auto account creation
  • 2FA with TOTP (Time-based One-Time Password)
  • OTP email verification (3 min TTL)
  • BCrypt password hashing
  • 10+ custom exception types
  • Cookie-based refresh (XSS protection)
